1. Getting Started & Setup

Python is an interpreted language: you write .py files and run them using the Python interpreter. Execution starts from the top of the script.

# Check Installation
# Windows: python --version  | Linux/Mac: python3 --version

print("Hello, World!") # Your first program

Interactive Mode (REPL)

Type python or py in your terminal to enter the shell (>>>). Use exit() to quit.

2. Basic Syntax & Identifiers

Python uses indentation (standard is 4 spaces) to define code blocks instead of {}.

if True:
    print("Correct Indentation")
# print("Error") # Missing indentation → IndentationError

Rules for Identifiers

  • Can contain letters (a–z), digits (0–9), and _.
  • Cannot start with a digit (e.g., 2name is invalid).
  • Cannot be a keyword (e.g., if, while, class).
  • Case-sensitive: age, Age, and AGE are different.

3. Variables, Naming & Scope

# Multi-assignment & Unpacking

Scope & global

count = 0
def increment():
    global count # Refer to variable in global scope

8. Lists (Mutable Sequence)

Ordered, indexable, and heterogeneous.

nums = [1, 2, 3]
nums.append(4)
nums.insert(1, "two") # Heterogeneous allowed
nums.pop()            # Remove & return last
nums.sort()           # In-place sort (only if items are same type)

List Comprehension

# [expression for item in iterable if condition]
vowels = [x for x in "education" if x in "aeiou"]

9. Tuples (Immutable Sequence)

Ordered and unchangeable. Often used for faster data access or to protect data.

t = (1, 2, 3)
single = (5,) # Comma is mandatory for single-element tuple

10. Sets & Frozensets

Unordered collection of unique, hashable elements.

s = {1, 2, 2, 3} # Result: {1, 2, 3}
s.add(4)
s.discard(10)    # Safe remove (no error if not found)

Frozenset: Immutable version of a set. Useful as a dictionary key.

11. Dictionaries (Key:Value)

Keys must be immutable (hashable). In Python 3.7+, insertion order is preserved.

student = {"name": "Anu", "age": 21}
print(student.get("phone")) # Returns None (Safe)
# .keys(), .values(), .items()

12. Conditionals, Match & Truthiness

# Shorthand (Ternary)

# Match Statement (3.10+)
match status:

14. Functions & Recursion

If no return is specified, a function returns None.

def factorial(n):
    if n <= 1: return 1 # Base Case
    return n * factorial(n - 1) # Recursive Case

16. The Four Pillars of OOP (Bank Example)

A BankAccount system is the perfect way to visualize how the four pillars work together in a real-world application.

from abc import ABC, abstractmethod

class BankAccount(ABC):
    def __init__(self, account_number, owner):
        self.account_number = account_number
        self.owner = owner
        self.__balance = 0.0  # Private attribute (Encapsulation)

    def get_balance(self):
        return self.__balance

    def deposit(self, amount):
        if amount <= 0: raise ValueError("Must be positive")
        self.__balance += amount

    @abstractmethod
    def withdraw(self, amount):
        """Abstraction: Each account defines its own rules."""
        pass

    def _deduct(self, amount):
        """Protected helper for subclasses."""
        if amount > self.__balance: raise ValueError("Funds insufficient")
        self.__balance -= amount

class SavingsAccount(BankAccount): # Inheritance
    def withdraw(self, amount):
        # Polymorphism: No overdraft allowed
        self._deduct(amount)

class CurrentAccount(BankAccount): # Inheritance
    def withdraw(self, amount):
        # Polymorphism: Allow overdraft up to 5000
        if (self.get_balance() - amount) < -5000:
            raise ValueError("Overdraft limit exceeded")
        self._deduct(amount)

1. Abstraction (The "What", not the "How")

Idea: Show only what the user needs; hide the internal complexity.

  • In the code: The caller just uses acc.withdraw(200). They don't need to know the math behind overdraft limits or internal balance storage.
  • Enforcement: Using ABC and @abstractmethod forces subclasses to implement specific logic while keeping the interface simple.

2. Encapsulation (Data Hiding)

Idea: Bundle data and methods; restrict direct access to data.

  • In the code: We use self.__balance. External code cannot do acc.__balance = 1000000.
  • Control: Updates are only possible via deposit(), which ensures all data follows validation rules (e.g., no negative deposits).

3. Inheritance (Code Reuse)

Idea: Reuse and extend behavior from a base class to reduce duplication.

  • In the code: SavingsAccount and CurrentAccount inherit account_number and deposit() from BankAccount.
  • Efficiency: Common logic is written once in the parent and shared across all child types.

4. Polymorphism (Many Forms)

Idea: Use the same method name to perform different actions based on the object type.

  • In the code: Both accounts have a withdraw() method. However, SavingsAccount.withdraw() blocks negative balances, while CurrentAccount.withdraw() allows them.
  • Flexibility: A single function can process any account type without knowing its specific rules in advance.

17. Decorators, Methods & Dunder

# Methods: @classmethod (cls), @staticmethod (none)
# Dunder: __str__ (string rep), __add__ (operator overloading)

class Point:
    def __add__(self, other):
        return Point(self.x + other.x, self.y + other.y)

Decorators: Functions that wrap and extend the behavior of another function.
